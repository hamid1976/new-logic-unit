# Master Milestones

## Phase 0: Source And Technical Baseline

Timeline: Week 1

Goal: make sure the site can be indexed and the team has one source of truth.

Tasks:

- Confirm all live pages and routes.
- Fix `noindex, nofollow` on production pages.
- Set self-referencing canonicals.
- Confirm sitemap uses production URLs.
- Confirm robots.txt points to production sitemap.
- Confirm metadata uses `www.logic-unit.com`.
- Confirm JSON-LD uses production URL values.
- Set up or verify Google Search Console.
- Set up or verify GA4.
- Define owner for content, design, frontend, and SEO QA.

Success criteria:

- Production pages are indexable.
- Sitemap and canonicals are correct.
- Website build tracker is active.

## Phase 1: Core Website Rebuild

Timeline: Weeks 2-5

Goal: rebuild the five core pages with SEO-ready copy, clear CTAs, and proper internal links.

Pages:

- Home
- About
- Industry
- Partnerships
- Contact

Tasks:

- Build reusable layout components.
- Build hero section component.
- Build CTA section component.
- Build card/grid components.
- Add page metadata.
- Add page schema.
- Add internal links.
- Add responsive QA.
- Add accessibility QA.

Success criteria:

- All core pages are developed and reviewed.
- Each page has one clear H1.
- Each page has metadata, canonical, CTA, and internal links.
- No footer/navigation labels are incorrectly used as main page H2s.

## Phase 2: Industry Expansion Pages

Timeline: Weeks 6-9

Goal: build source-derived industry support pages.

Pages:

- Retail Software Platforms
- Healthcare Software Platforms
- Livestock Software Platforms
- Manufacturing Software Platforms
- Logistics Software Platforms
- Facilities Software Platforms

Tasks:

- Build industry detail page template.
- Add each industry page.
- Link from `/industry/`.
- Link relevant homepage industry labels to `/industry/`.
- Add breadcrumb links.
- Keep unsupported details marked for later source approval.

Success criteria:

- Six industry pages are live or ready for approval.
- Industry page links to each detail page.
- Each industry page has metadata and CTA.

## Phase 3: Partnership Expansion Pages

Timeline: Weeks 10-12

Goal: build commercial-intent pages for partnership routes.

Pages:

- Strategic Partnerships
- System Integrators
- Enterprise Collaboration

Tasks:

- Build partnership detail page template.
- Add each partnership route page.
- Link from `/partnerships/`.
- Link to `/contact/`.
- Align form inquiry types with page CTAs.

Success criteria:

- Three partnership detail pages are live or ready for approval.
- Partnership funnel is clear.
- Contact form inquiry types match CTA language.

## Phase 4: Trust And Proof Layer

Timeline: Months 4-6

Goal: make the company more credible without unsupported claims.

Tasks:

- Add leadership section if approved.
- Add company timeline if approved.
- Add product ecosystem page if approved.
- Add product screenshots or diagrams.
- Add customer proof only if source-approved.
- Add privacy/terms pages if required.
- Add careers/talent page if hiring is a priority.

Success criteria:

- Site has stronger EEAT signals.
- Partnership and enterprise audiences have more confidence.
- No unsupported proof claims are published.

## Phase 5: Content And Thought Leadership

Timeline: Months 7-12

Goal: build topical authority.

Content themes:

- What is a SaaS operating company?
- Product engineering for industry SaaS platforms
- Building software platforms for real industries
- Industry-specific SaaS platforms
- SaaS architecture for operational industries
- System integrator partnerships for SaaS platforms
- Product ecosystem building

Success criteria:

- Monthly content cadence exists.
- Search Console shows growing impressions.
- Core pages gain internal links from content.

## 12-Month Marketing Direction

Quarter 1:

- Fix technical SEO.
- Launch core website.
- Establish positioning.

Quarter 2:

- Launch industry and partnership pages.
- Add trust/proof layer.
- Begin targeted outreach.

Quarter 3:

- Launch thought leadership.
- Build partner-focused campaigns.
- Improve conversion tracking.

Quarter 4:

- Publish ecosystem updates.
- Review search performance.
- Expand only where data supports it.
