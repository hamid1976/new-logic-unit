# Logic Unit Website Build Plan

Purpose: turn the CEO-level direction into a clear website development plan that the team can execute, track, and ship.

Core positioning:

Logic Unit builds and operates industry-specific SaaS platforms for real industries.

Primary website goal:

Make Logic Unit commercially clear, technically credible, SEO-ready, and conversion-ready for partners, enterprises, system integrators, investors, and talent conversations.

## Non-Negotiables

- Do not present Logic Unit as a generic software agency.
- Do not create unsupported service pages such as web development, mobile app development, ERP, CRM, or IT outsourcing.
- Fix `noindex, nofollow` before launch.
- Use self-referencing canonical URLs.
- Each page needs a clear CTA.
- Each page needs defined keywords, H1, metadata, sections, internal links, and image requirements.
- Any proof claims, customer names, case studies, awards, offices, certifications, or metrics require source approval.

## Recommended Launch Order

1. Technical SEO foundation
2. Homepage
3. About
4. Industry
5. Partnerships
6. Contact
7. Industry detail pages
8. Partnership detail pages
9. Product ecosystem pages, only when product source data is ready
10. Content hub / insights, only after core pages are complete
