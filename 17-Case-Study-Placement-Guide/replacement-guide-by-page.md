# Replacement Guide By Page

This guide tells the developer what should change or be replaced on each current page.

## Global Navigation And Footer

### Add

- Add `Case Studies` navigation item linking to `/case-studies`.
- Add footer link under Company or Resources: `Case Studies`.

### Replace

- If any case study cards currently point to `/contact` only, replace those links with individual detail-page links.
- Keep `/contact` as the CTA inside each detail page, not as the card destination.

## Individual Case Study Pages

### Add

Each page should use this structure:

1. Breadcrumb
2. Hero with industry label, H1, short summary, and CTA
3. Client or context overview
4. Business challenge
5. Logic Unit solution
6. Key features
7. Technology highlights
8. Business impact or results
9. Related case studies
10. Final CTA

### Replace

- Replace any generic "project details coming soon" copy with the full page copy from `16-Products-CaseStudies-Content/03-Case-Study-Pages`.
- Replace placeholder CTAs with `Contact Logic Unit`.
- Replace unsupported metric blocks with only source-approved results.
