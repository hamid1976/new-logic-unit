# Case Study Page Enhancement Template

Use this to strengthen every case study page.

## Required Structure

1. Hero
2. Executive summary
3. Client context
4. Business challenge
5. Logic Unit solution
6. Key features
7. Technology highlights
8. Results or business impact
9. Related platform/product links
10. CTA

## Add To Each Case Study

### Technology Highlights

Use source-backed technology only. Do not invent tools or stack items.

### Business Impact

Use qualitative impact when numeric metrics are not available.

Acceptable:

- improved visibility
- reduced manual workflows
- improved customer self-service
- centralized operations
- stronger reporting

Avoid:

- percentage improvements unless source provides them
- revenue claims
- unverified download counts
- client testimonials not in source

### Visual Module

Add one visual per case study:

- workflow diagram
- before/after process
- platform module map
- dashboard-style mockup
- project category visual

### CTA

Use page-specific CTA:

- Logistics: `Discuss A Logistics Platform`
- Healthcare: `Discuss A Healthcare Platform`
- Retail: `Discuss A Retail Operations Platform`
- Government: `Discuss A Government Digital Platform`
- Enterprise: `Discuss Enterprise Collaboration`
